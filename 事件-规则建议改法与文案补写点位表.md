# 《赤壁前夜》事件 / 规则建议改法 + 文案补写点位表

## 0. 这份表的用途

- 这份文档只做整理，不直接改玩法代码。
- 目标是把当前仓库里已经确认的问题、建议改法、文案缺口和落点文件先归拢清楚。
- 每个部分都留了可填写空档，方便你直接写“我希望怎么改”“我希望什么文风”“先做哪几项”。
- 原则仍然保持：**尽量局部、小改、不破坏前 5 步教学，不把玩法逻辑继续堆进 UI。**

---

## 1. 当前系统状态速记

- 事件定义入口：`scripts/core/game_data.gd`
- 事件触发 / 判定 / 过期入口：`scripts/managers/event_manager.gd`
- 回合结算与结果串联：`scripts/managers/turn_manager.gd`
- 教学流程入口：`scripts/managers/tutorial_manager.gd`
- 事件标题与简介文本：`data/texts/events.json`
- 通用日志模板：`data/texts/logs.json`
- 回合汇报文本：`data/texts/turn_report.json`
- 当前事件文本主要还是 `title + description`，**缺少每个事件自己的成功 / 失败 / 过期叙事文本**
- 当前已确认有几处“定义已写，但规则没真正接上”的情况
