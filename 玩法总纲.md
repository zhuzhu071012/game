# 《赤壁前夜》v0.1 玩法总纲

## 1. 项目定位

《赤壁前夜》是一款 **2D 桌面式可拖动卡牌叙事策略游戏**。
玩家扮演赤壁前一年的曹操，在有限时间内通过 **治事、召对、案读、访求、入寐** 五种日常手段，处理军政、招揽人才、研究情报、控制局势、压制隐患，并应对按时令或条件不断弹出的突发事件，尽量避免局势滑向赤壁之火与多种坏结局。

本作的核心不是“统一天下”，而是：

**玩家面对持续压力，充分协调有限的资源，并尽量创造自己的优势。**

------

## 2. 核心体验

v0.1 要验证的核心体验有三条：

### 2.1 高压管理感

玩家每回合都需要在有限人物、有限资源、有限时间中做选择。
很多问题不能同时解决，必须取舍。

### 2.2 风险累积感

失败不是简单掉数值，而是会转化为：

- 隐患牌累积
- 关系变坏
- 后续事件变强
- 重要角色死亡或脱离队伍
- 终局评价下降

### 2.3 从孤立到成势的成长感

开局只有玩家一人，手下和资源需要通过游戏操作逐步获得。
玩家会经历从“孤身应付”到“逐渐拥有自己的班底”的过程。

------

## 3. 一句话核心循环

**投入卡牌 → 点击结束回合 → 统一结算常驻槽位与事件 → 推进时令 → 刷新新事件与新选择。**

------

## 4. v0.1 范围定义

### 4.1 v0.1 必须成立的东西

- 5 个常驻功能槽能正常运作
- 事件能按时间或条件弹出
- 玩家能通过人物与资源解决事件
- 失败会带来真实代价
- 隐患牌 3 张坏结局机制成立
- 一局能从开局打到终局
- 玩一局后可以判断：
  - 哪个槽位有用
  - 哪种资源紧张
  - 哪类事件有意思
  - 哪条失败路线最常出现

### 4.2 v0.1 暂不深做的内容

- 深度爱情线
- 复杂嫉妒与私情暴露系统
- 深层俘虏经营
- 多地区战略地图
- 派系政治
- 大规模剧情演出
- 高质量正式美术

------

## 5. 核心系统总览

v0.1 的主要系统如下：

- 常驻功能槽系统
- 事件系统
- 人物系统
- 资源系统
- 关系系统
- 隐患牌系统
- 回合结算系统
- 终局战斗系统

这些系统共同组成主循环，不以单一剧情推进为核心，而以“系统压力与选择代价”推动体验。

------

## 6. 时间表现原则

v0.1 在表现层使用**节气**作为时间标识，以增强时令感与历史氛围。
内部规则层采用固定可操作回合数，以便验证玩法密度与单局节奏。

### 时间结构

- **可操作阶段：32 回合**
- **时间范围：建安十二年秋分 → 建安十三年小寒**
- **第 33 回合：不再操作，直接进入结局结算**

### 提前结算规则

在可操作的 32 回合内，如果触发**坏结局**，则提前进入结局结算。

### 火势规则

火势在 v0.1 中仅作为时令推进的外显进度条，用于表现赤壁节点不断逼近。
火势不承担失败判定与坏结局功能。

------

## 7. 五个常驻功能槽

五个常驻功能槽是本作主循环的核心。
其主要作用不是直接推进剧情，而是通过**产出资源、消耗资源、转化状态**来支撑整局运转。

在 v0.1 中，五个槽位分别承担以下职责：

- **治事**：基础经济与政务维稳
- **召对**：关系经营与软性危机处理
- **案读**：研究、推演与情报转化
- **访求**：扩充班底与搜求外部机会
- **入寐**：恢复、疗养与损耗修复

### 7.1 资源产出与消耗循环

五个常驻功能槽的主要作用是产生资源和消耗资源。
v0.1 当前的基础资源循环如下：

## 8. 事件系统（等完善）

### 8.1 事件定义

事件不是常驻槽位，而是 **临时出现的一次性问题卡**。
事件按“时令节点”或“条件”弹出，玩家需要投入人物或资源解决。

### 8.2 事件来源

#### 时令触发

按外显时令、阶段节点或特定进程触发。例如：

- 某一节气到来
- 某一阶段开始
- 某类局势进入特定阶段

#### 条件触发

由当前状态引发。例如：

- 后方稳定过低
- 流言过高
- 特定关系值达到阈值
- 某前置事件完成

### 8.3 事件处理方式

事件弹出后：

- 玩家可向事件投入人物卡
- 也可投入资源卡
- 满足要求后可在回合结算中处理

事件处理完成后直接消失。

### 8.4 事件结果

#### 成功解决

- 事件消失
- 获得奖励
- 可能解锁新线索、新人物、新资源或改善全局值

#### 失败或放置不理

- 事件消失
- 生成隐患牌
- 或附带其他负面后果

## 9. 人物、资源、事件属性

### 9.1 属性体系总览

角色采用 **六维基础属性 + 专长标签** 的结构。

#### 六维基础属性

- **力量**：近身压制、武力执行、强行突破
- **敏捷**：反应、机动、灵活处理、快速应对
- **体质**：耐久、承压、持续行动能力、伤病恢复基础
- **智力**：研究、推演、知识处理、分析能力
- **感知**：观察、识人、判断、察觉异常
- **魅力**：说服、安抚、威慑表达、社会影响力

### 9.2 常驻槽位与属性对应

| 槽位 | 使用属性       | 说明                       |
| ---- | -------------- | -------------------------- |
| 治事 | 体质 + 智力    | 处理政务、军务、财赋、维稳 |
| 召对 | 感知 + 魅力    | 沟通、安抚、施压、关系处理 |
| 案读 | 智力 + 感知    | 研究、解读、推演、整理情报 |
| 访求 | 体质 + 魅力    | 寻访、招揽、搜求、外出奔走 |
| 入寐 | 不使用属性判定 | 作为恢复、疗养、治病手段   |

### 9.3 战斗 / 冲突事件属性

战斗或冲突型事件主要使用：

- **力量 + 敏捷**

根据事件类型，可以少量追加辅助属性：

- 体质：持续作战、硬扛
- 感知：识破埋伏、判断时机

### 9.4 专长标签体系

专长标签用于提供额外加成，或作为某些事件的必要资格。
专长**不替代属性**，而是在判定时叠加修正。

#### 当前专长标签

- **医疗**：疗伤、治病、处理疫气、压制病势
  - 特殊地位：部分事件只有医疗达标时才可解决
- **奇谋**：处理复杂局势、特殊方案、谋略事件
- **搜索**：找人、找物、寻线索、搜查
- **交涉**：说服、安抚、沟通、关系推进
- **欺诈**：掩饰、诈术、诱导、虚实操作
- **威慑**：压制、恐吓、强推局面
- **治军**：军纪、整训、统率、处理军中事务

### 9.5 判定体系总则

v0.1 不采用 DND 标准修正值比例，而是借用其“属性 + 专长 + 资源 + 掷骰”的判定结构。
为强化人物培养、资源准备与长期经营的作用，所有常规判定统一采用：

**原始属性 + 专长加成 + 资源加成 + 2d6**

也就是说：

- 角色基础属性直接参与判定，不先转换为修正值
- 专长提供固定加成
- 资源提供固定加成或资格
- 2d6 仅提供中等幅度波动，不喧宾夺主

### 9.6 属性判定规则

#### 基础公式

采用 **原始属性 + 2d6** 的判定体系：

**2d6 + 属性和 + 专长加成 + 资源修正 ≥ 事件难度值（DC）**

则事件成功。

#### 判定步骤

1. 确定事件调用的属性组合
2. 计算主处理人的属性和
3. 计算辅助角色属性和的一半，向下取整
4. 叠加专长加成
5. 叠加资源修正
6. 掷 **2d6**
7. 与事件 DC 比较

#### 最低要求

事件可设置**最低要求**。
若投入对象不满足最低要求，则该事件**不能成功解决**，只能失败或维持未解。

#### 辅助角色规则

- 主处理人 A 提供完整属性贡献
- 辅助角色 B 提供 0.5 倍属性贡献
- 若 B 空缺，则其贡献为 0

#### 数值说明

- 角色原始属性范围为 **0 到 5**
- 2d6 的结果范围为 **2 到 12**
- 由于角色原始属性直接参与判定，因此人物与资源在整体判定中占主要权重，随机项只提供有限波动

### 9.6.1 难度等级表

为统一事件、槽位与终局判定口径，v0.1 当前采用以下难度等级：

| 难度等级 | DC   | 说明                           |
| -------- | ---- | ------------------------------ |
| 极易     | 8    | 基础操作，基本不构成压力       |
| 简单     | 10   | 常规低风险任务                 |
| 普通     | 13   | 默认标准任务                   |
| 较难     | 16   | 需要合适人物或资源支撑         |
| 困难     | 19   | 需要较好准备才能稳定解决       |
| 极难     | 22   | 需要强人物、专长与资源共同支撑 |
| 决战级   | 25+  | 终局阶段或关键剧情判定使用     |

### 9.7 入寐规则

入寐不做属性判定。
它是一个**恢复与疗养系统**，主要通过投入对象与资源决定效果。

#### 入寐可投入

- 受伤角色
- 金钱，或
- 草药，或
- 具有“医疗”专长的人物

#### 入寐可产出

- 恢复属性
- 缓解伤病
- 缓解部分持续负面状态
- 推进相关人物事件接口

### 9.8 受伤与属性变化

角色属性分为两层：

#### 基础属性

角色原始能力，不轻易变化，仅作为数据存储。

#### 当前属性

受以下因素影响：

- 受伤
- 疾病
- 疲劳
- 事件惩罚
- 特殊状态

#### 属性降低规则建议

- **轻伤（战斗事件判定失败）**：相关属性 -1
- **重伤（战斗事件判定大失败）**：相关属性 -2，或两项属性各 -1
- **疾病**：优先影响体质，也可连带影响智力 / 感知
- **精神受挫**：可影响感知 / 魅力 / 智力

#### 属性恢复方式

- **入寐恢复**：主要恢复手段

### 9.9 资源对属性判定的作用

资源不仅构成产出与消耗循环，也会在事件判定、槽位结算与终局准备中提供修正、资格或恢复效果。

| 资源     | 固定加成/效果                                                | 备注                                   |
| -------- | ------------------------------------------------------------ | -------------------------------------- |
| 银两     | 每投入 **1 份银两 = +1** 通用修正，单次事件或槽位结算最多计 **+2** | 通用资源，适用最广，但上限低，避免万能 |
| 草药     | 疫病事件中投入草药后 **+2**                                  | 疫病必须有草药，否则不能成功解决       |
| 密报     | 军略、奇谋、诈降相关判定中每 1 份密报 **+2**，单次最多计 **+4** | 偏高价值情报资源                       |
| 求贤文书 | 不提供数值加成，但提供“进入人才结果池资格”                   | 资格资源                               |
| 礼物     | 普通礼物 **+2** 召对修正；若礼物类型与目标角色偏好匹配，则 **+3** | 礼物一次只能放 1 个                    |
| 逍遥     | 头风相关事件中投入可直接解除一张头风牌                       | 精神恢复专用资源                       |
| 舟图     | 水事、火攻变局、终局水战相关判定中每 1 份舟图 **+3**，单次最多计 **+6** | 稀缺战备资源，价值高                   |

### 9.10 事件类别与属性调用模板

| 事件类别      | 主要属性                                 | 推荐专长         |
| ------------- | ---------------------------------------- | ---------------- |
| 战斗 / 冲突类 | 力量 + 敏捷                              | 治军、威慑       |
| 治事 / 后勤类 | 体质 + 智力                              | 治军             |
| 召对 / 人心类 | 感知 + 魅力                              | 交涉、威慑、欺诈 |
| 案读 / 研究类 | 智力 + 感知                              | 奇谋、搜索       |
| 访求 / 搜索类 | 体质 + 魅力                              | 搜索、交涉       |
| 疾病 / 疗伤类 | 不走常规属性判定，优先走入寐恢复逻辑     | 医疗             |
| 水事 / 晕舟类 | 力量 + 敏捷 或 智力 + 感知（视事件性质） | 治军、奇谋       |

## 10. 人物系统

### 10.1 开局结构

| 张辽 | 4    | 5    | 4    | 2    | 3    | 3    | 搜索             |
| 于禁 | 4    | 3    | 4    | 2    | 3    | 1    | 治军             |

## 11. 关系系统

### 11.1 v0.1 范围

当前关系系统采用轻量版，只验证以下内容：

- favor
- rumor_risk
- 基础关系档位
- 至少一条关系事件链

### 11.2 关系系统的作用

关系系统不只是数值装饰，它应实际影响：

- 召对收益
- 某些事件结果
- 流言压力
- 未来扩展角色个人线的接口

### 11.3 v0.1 不深做的部分

- 深度角色个人线
- 嫉妒链
- 私情暴露主线
- 多角色互相关系网

这些内容只保留接口，不作为当前版本重点。

## 12. 隐患牌系统

### 12.1 系统定位

隐患牌是本作最重要的失败累积机制之一。
它们不是普通 debuff，而是 **具体坏结局的门票**。

### 12.2 v0.1 五类隐患

- 头风
- 离心
- 疫气
- 流言
- 晕舟

### 12.3 规则

- 同类隐患累计 3 张，直接触发对应坏结局，分别为：
  - 神离
  - 众散
  - 疫溃
  - 谤成
  - 覆舟
- 部分隐患可通过高代价手段缓解或移除

### 12.4 隐患设计目标

玩家应始终知道：

- 自己现在正在往哪种坏结局滑去
- 哪些行为在加剧隐患
- 哪些手段可以暂时压住风险

这会让失败具有方向感和叙事感，而不是单纯数值归零。

## 13. 回合与结算框架

### 14.1 结算规则

| 结算对象   | 何时结算              | 结果类型                          | 规则说明                                                     |
| ---------- | --------------------- | --------------------------------- | ------------------------------------------------------------ |
| 常驻槽位   | 回合末                | 收益 / 恢复 / 产出                | 所有常驻槽位在回合末统一结算，结算顺序固定为：**治事 → 召对 → 案读 → 访求 → 入寐** |
| 事件       | 回合末                | 成功 / 失败 / 未解                | 每个事件具有属性要求与时限要求。投入人物和资源后，在回合末统一判定。若在限定时限内未被解决，则记为未解，效果等同失败。普通事件默认时限为 3 回合；紧急事件默认时限为 1–2 回合；长期事件默认时限为 5 回合以上 |
| 事件判定   | 回合末                | 必定成功 / 掷骰成功 / 失败 / 未解 | 每个事件具有最低要求、成功阈值与时限。当投入对象的原始属性总和达到成功阈值时，事件必定成功；当达到最低要求但未达到成功阈值时，进入 2d6 判定；当低于最低要求时，事件不能成功解决，只会失败或维持未解 |
| 个人线情况 | 回合末                | 推进 / 恶化 / 缓解                | 角色个人线在回合末统一检查。若关系值、状态值或其他特定条件达到阈值，则触发个人线推进、恶化或缓解 |
| 隐患牌     | 即时检查 + 回合末惩罚 | 惩罚 / 坏结局                     | 隐患牌在生成、移除或变化时即时检查。若同类隐患累计达到 3 张，立即触发对应坏结局，游戏结束。若未达到 3 张，则其层数惩罚在回合末统一结算 |
| 终局       | 第 33 回合或提前触发  | 好 / 中 / 坏结局                  | 前 32 回合为可操作回合。第 33 回合不再进行新操作，只执行终局判定。若坏结局成立，则提前进入结局结算 |

### 13.2 事件判定具体说明

| 项目           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 判定核心       | 采用 **原始属性 + 2d6** 的判定思路                           |
| 事件输入       | 人物原始属性、资源修正、特殊加成                             |
| 最低要求       | 未达到最低要求时，事件无法成功，只能失败或维持未解           |
| 成功阈值       | 达到成功阈值时，事件直接成功，不需要掷骰                     |
| 掷骰区间       | 达到最低要求但未达到成功阈值时，进入 **2d6** 掷骰判定        |
| 掷骰公式建议   | **2d6 + 原始属性和 + 资源修正 + 特殊修正 ≥ 事件难度值（DC）** 则成功 |
| 高结果（可选） | 若 2d6 结果极高，可额外获得奖励或避免副作用                  |
| 低结果（可选） | 若 2d6 结果极低，可触发额外惩罚或追加隐患                    |
| 未解           | 事件时限归零仍未解决，则记为未解，按失败处理                 |
| v0.1 建议      | 第一版先做“普通成功 / 失败 / 未解”，高低结果先留接口，不必立刻做深 |
| 主处理人       | 每个事件必须指定 1 名主处理人，使用其主要属性作为核心修正来源 |
| 辅助人物       | 其他投入人物提供辅助修正，按 0.5 倍属性贡献并向下取整        |
| 稳过阈值       | 当总值达到成功阈值时，可直接成功，避免高配局仍过度看脸       |
| 最低要求判定   | 最低要求优先于掷骰；不满足最低要求时，连掷骰资格都没有       |

### 13.3 常驻槽位判定补充说明

#### 主处理人与辅助处理人

- 每个常驻槽位默认有 1 名主处理人 A
- 可选 1 名辅助处理人 B
- 辅助处理人的属性贡献统一按 0.5 倍计算
- 若 B 空缺，则其贡献记为 0

#### 判定统一原则

- 治事、召对、案读、访求均可使用 2d6 作为随机项
- 入寐不进行常规属性判定，而采用直接恢复规则
- 所有涉及除法的资源产出结果均向下取整
- 所有资源产出最低为 0，不出现负值

## 14. 终局规则

终局主要由**阵营属性对比**与**分阶段战斗判定**共同决定。
玩家在前 32 回合中积累的资源、人物、事件成果与关系状态，将沉淀为以下阵营属性：

- 粮草
- 兵力
- 人心
- 军略

第 33 回合不再进行新操作，而是进入终局结算。
终局战斗分为四个阶段：

1. 战前整备
2. 火攻变局
3. 诈降与机谋
4. 正面决战

各阶段根据阵营属性、资源、人物与关键事件状态进行判定，并共同累积**胜利值**。

### 14.1 阵营属性表

| 阵营属性 | 含义                             | 主要来源                         | 终局中的作用                           |
| -------- | -------------------------------- | -------------------------------- | -------------------------------------- |
| 粮草     | 粮秣、后勤、军需保障能力         | 治事、后勤事件、资源积累         | 影响战前整备与最终决战                 |
| 兵力     | 军队总量、战力完整度、损耗控制   | 招揽武将、军队资源牌、战斗前积累 | 影响火攻后承受能力与最终决战           |
| 人心     | 内部稳定、士气、归附程度、信任感 | 召对、流言控制、关系经营         | 影响战前状态与最终决战                 |
| 军略     | 谋略、情报、预判、识破敌计的能力 | 案读、谋士、密报、关键事件成功   | 影响火攻变局与诈降判定，也影响最终决战 |

### 14.2 终局四阶段规则表

| 阶段 | 名称       | 主要输入                                 | 判定目标                                   | 成功结果               | 失败结果           | 对胜利值影响                                        |
| ---- | ---------- | ---------------------------------------- | ------------------------------------------ | ---------------------- | ------------------ | --------------------------------------------------- |
| 1    | 战前整备   | 粮草、人心                               | 判断战前状态是否稳固                       | 战前状态稳固           | 战前状态不稳       | 粮草达标则 +1，不达标则 -1；人心过低可额外 -1       |
| 2    | 火攻变局   | 军略、密报、相关事件状态                 | 是否看透铁索连环的风险，并提前掌握风向变化 | 成功避开或减轻火攻冲击 | 火攻成功重创己方   | 成功无变化；失败兵力 -2；大失败兵力 -3 且胜利值 -1  |
| 3    | 诈降与机谋 | 军略、谋士、密报、密信等                 | 是否识破诈降与敌方机谋                     | 识破敌计，占得先机     | 误判局势，失去先机 | 成功 +1；大成功额外使第四阶段 +1；失败 0；大失败 -1 |
| 4    | 正面决战   | 粮草、兵力、人心、军略、武将、军队资源牌 | 判断最终正面交锋的结果                     | 获得战场优势           | 战场失利           | 成功 +2；大成功 +3；失败 -2；大失败 -3              |

### 14.3 胜利值分档表

| 胜利值区间 | 战斗结果 | 大结局基调      |
| ---------- | -------- | --------------- |
| 4 ~ 5      | 大胜     | 相对好结局      |
| 2 ~ 3      | 胜       | 普通偏好结局    |
| -1 ~ 1     | 平       | 中性 / 勉强维持 |
| -2 ~ -3    | 败       | 普通偏坏结局    |
| -4 ~ -5    | 大败     | 坏结局主基调    |

### 14.4 人物结局层

每个关键人物在终局时均根据以下因素生成个人结局：

- 是否加入
- 是否存活 / 离队
- 当前关系状态
- 个人线推进情况

人物结局不改写大结局结果，而作为终局尾声，对本局叙事进行补充。

## 15. 一局的结束条件

v0.1 一局结束主要来自以下几类：

### 15.1 时令走尽

游戏在表现层以节气推进。
当外显时令推进至终局节点时，进入最终结算。

### 15.2 立即结局

- 某类隐患牌累计 3 张（坏结局）

### 15.3 终局结算

- 时令走尽或触发立即结局后，根据数值判断进入好结局、普通结局或坏结局
